## What was built

### Project structure (`/home/user/app`)
```
index.html        ← Simple page with "Verify Identity" button + result box
src/
  main.js         ← PremblyPass widget class + SDK initialisation
  style.css       ← Base styles + modal/overlay/result-box styles
.env              ← VITE_PREMBLY_API_KEY loaded from $PREMBLY_API_KEY
```

### `PremblyPass` widget class (`src/main.js`)
A custom class built on top of the `prembly-pass` SDK that:

| Feature | Detail |
|---|---|
| **Initialisation** | `app_id: 'test_app_id'`, `x_api_key` from `VITE_PREMBLY_API_KEY`, `environment: 'test'`, `base_url: 'https://api.prembly.com'` |
| **Config** | `setup({ config_id: 'test_config_id', user_ref: '<unique>' })` — a new `user_ref` is generated on every button click |
| **Widget launch** | `pass.open()` renders a full-screen backdrop + modal iframe pointing to `https://api.prembly.com/pass/widget?...` |
| **Events** | Listens for `postMessage` from the iframe (`PREMBLY_PASS_SUCCESS`, `PREMBLY_PASS_ERROR`, `PREMBLY_PASS_CLOSE`); emits `success`, `error`, and `close` events |
| **Logging** | `success` payload is `console.log`-ged; `error` payload is `console.error`-ed; both are also rendered on-screen in a colour-coded result box |

### API key
The environment variable `PREMBLY_API_KEY` is forwarded to Vite as `VITE_PREMBLY_API_KEY` via `.env`. In `main.js` it is read with `import.meta.env.VITE_PREMBLY_API_KEY`.
