## How It Works

**`index.js`** creates an Express server that:

1. **Matches** any method on `/api/prembly/*path`
2. **Strips** the `/api/prembly` prefix and forwards the remaining path to `https://api.prembly.com`
3. **Injects** `app-id` and `x-api-key` headers from environment variables — credentials never touch the frontend
4. **Forwards** the original HTTP method, request body, and query parameters via `axios`
5. **Proxies error responses** from Prembly back to the caller with the original status code

## Starting the Server
```bash
PREMBLY_APP_ID=your_app_id PREMBLY_API_KEY=your_api_key node index.js
```
Or export the variables in your environment / `.env` loader before running `node index.js`.

## Example Frontend Usage
Instead of calling Prembly directly (which would expose credentials), the frontend calls:
```
POST http://localhost:3000/api/prembly/identitypass/verification/nin
```
The proxy transparently forwards this to:
```
POST https://api.prembly.com/identitypass/verification/nin
```
with the secret headers injected server-side.
