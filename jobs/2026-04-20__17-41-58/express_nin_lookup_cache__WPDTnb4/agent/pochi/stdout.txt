## How it works

**`index.js`** sets up:
- A `Map` (`ninCache`) as the in-memory cache keyed by NIN string.
- `POST /api/verify-nin` endpoint that:
  1. Validates the `nin` field is present in the request body.
  2. Checks `ninCache` — if a cached entry exists, returns it immediately with `{ cached: true, data: ... }`.
  3. Otherwise, calls the Prembly sandbox API (`https://api.prembly.com/verification/nin`) with the required `app-id` and `x-api-key` headers sourced from `PREMBLY_APP_ID` and `PREMBLY_API_KEY` environment variables.
  4. On success, stores the response in `ninCache` and returns `{ cached: false, data: ... }`.
  5. On failure, proxies the Prembly error status and body back to the client.
