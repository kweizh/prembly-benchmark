## What was built

### `POST /api/wallet/create`

| Step | Detail |
|------|--------|
| **Input validation** | Rejects with `400` if `ninNumber` or `faceLivelinessRef` are missing |
| **Credential guard** | Returns `500` if `PREMBLY_APP_ID` / `PREMBLY_API_KEY` env vars are absent |
| **NIN verification** | Calls `POST https://api.prembly.com/verification/nin` with `{ number: ninNumber }` and headers `app-id` / `x-api-key` |
| **Result check** | Accepts only when Prembly responds with `status: true` **and** `verification.status: "VERIFIED"` |
| **Success** | `201 { "wallet_id": "new_wallet_123" }` |
| **Failure** | `400 { "error": "<reason>" }` |
| **Upstream error** | `500 { "error": "<reason>" }` |

A `GET /health` endpoint is also included for liveness probes.
