### How it works

`POST /api/verify-nin` accepts:
```json
{ "number": "<NIN>" }
```

It then makes a proxied POST request to `https://api.prembly.com/api/v1/verification/nin` with:
- `app-id` header → `PREMBLY_APP_ID` env var
- `x-api-key` header → `PREMBLY_API_KEY` env var

The Prembly response (status code + JSON body) is forwarded directly to the client.

### Error handling
| Scenario | HTTP Status | Response |
|---|---|---|
| Missing `number` field | `400` | `{ "error": "Missing required field: number" }` |
| Missing env credentials | `500` | `{ "error": "Server configuration error: missing API credentials" }` |
| Unexpected exception | `500` | `{ "error": "Internal server error", "message": "..." }` |
| Prembly error | Mirrors Prembly's status | Mirrors Prembly's JSON body |

### Usage
Set environment variables before starting:
```bash
PREMBLY_APP_ID=your_app_id PREMBLY_API_KEY=your_api_key npm start
```

Example request:
```bash
curl -X POST http://localhost:3000/api/verify-nin \
  -H "Content-Type: application/json" \
  -d '{"number": "12345678901"}'
```
