## Key Features Implemented

### 1. Express Server Setup
- Server runs on `process.env.PORT` or defaults to port 3000
- JSON body parsing middleware enabled
- Start command: `npm start`

### 2. POST /verify-nin Endpoint
- Accepts JSON body with `nin` string (e.g., `{"nin": "12345678901"}`)
- Validates that NIN is provided
- Returns 400 if NIN is missing

### 3. Exponential Backoff Retry Mechanism
- **Max retries**: 3
- **Retry delays**: 100ms → 200ms → 400ms (exponential backoff)
- Only retries on 429 status codes
- Other errors fail immediately
- If all retries exhausted, returns 429 to client with message: `"Rate limit exceeded, please try again later"`

### 4. Prembly API Integration
- Base URL: `process.env.PREMBLY_BASE_URL` (defaults to `https://api.prembly.com`)
- Path: `/verification/nin`
- Headers: `app-id` and `x-api-key` from environment variables

### 5. Error Handling
- 400: Missing NIN parameter
- 200: Successful verification (returns Prembly API response)
- 429: Rate limit exceeded (after 3 retries)
- 500: Other errors
